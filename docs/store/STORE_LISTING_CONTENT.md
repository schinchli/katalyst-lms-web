## Screenshots — Exact Play Store Specs

### Phone (required — minimum 2, maximum 8)

| Spec | Value |
|------|-------|
| Format | JPEG or 24-bit PNG (no alpha channel) |
| Recommended size | **1080 × 1920 px** (9:16 portrait) |
| Modern phones | 1080 × 2400 px (20:9) also accepted |
| Minimum | 320px on shortest side |
| Maximum | 3840px on any side |
| Aspect ratio | Between 16:9 and 9:16 |

> **Simulator output:** iPhone 16 Pro screenshots from `xcrun simctl` come out at **1179×2556** (3× scale). These are valid — Play Store accepts any size within limits. You can use them directly or downscale to 1080×2340.

### Android Emulator for native screenshots
```bash
# Pixel 9 outputs at 1080×2424 — perfect for Play Store
# After connecting emulator: adb exec-out screencap -p > screenshot.png
```

### Feature Graphic (required if you have a promo video, shown as thumbnail)
| Spec | Value |
|------|-------|
| Size | **1024 × 500 px** |
| Format | JPEG or 24-bit PNG (no alpha) |
| Safe zone | Keep text/logos within centre 924×400 (50px padding each side) |

### App Icon (Play Store listing icon, separate from in-app icon)
| Spec | Value |
|------|-------|
| Size | **512 × 512 px** |
| Format | 32-bit PNG (with alpha OK here) |
| Source | Export from `mobile/assets/images/icon.png` (1024×1024) |
